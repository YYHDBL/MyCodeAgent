核心特性

Reasoning

Function Calling 驱动

严格结构化的工具调用协议，让每一步行动可验证、可恢复，模型切换时仍保持稳定。

Context

层级上下文工程

系统规则、项目规范、历史轮次分层组织，压缩与截断可控，长链路也不崩窗。

Observability

全链路可观测

结构化日志 + HTML Trace，任务执行路径一眼可见，便于复盘与调试。

Tooling

12+ 内置工具

读写、搜索、Patch、Todo、Shell 全覆盖，日常开发动作一站式完成。

Skills

技能插件化

按需注入的 Skills Prompt，让 Agent 在关键领域快速“上手”。

MCP

外部能力扩展

通过 MCP 接入外部工具生态，能力边界可持续扩张。

架构流程

01 / INPUT

结构化消息

用户需求 → 规则 → 记忆，按层构建上下文。

02 / THINK

策略编排

模型按工具协议输出计划与调用。

03 / ACT

工具执行

本地与 MCP 工具统一执行与校验。

04 / TRACE

可观测记录

Trace 记录完整链路，支持回放与分析。

MyCAT 关注“可解释 + 可复盘 + 可扩展”，让 AI 编程助手更像可靠的工程伙伴。
