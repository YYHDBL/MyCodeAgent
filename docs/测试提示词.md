## Prompt 1（基础能力：探索+理解+小规模产出）

  提示词：

  请使用subagent快速理解这个仓库是做什么的，并输出一份“项目概览”文档：
  - 简要说明目标、核心模块、主要功能；
  - 引用真实文件路径作为依据；
  - 输出一个结构清晰的 Markdown 文件放在 tests/demo/ 项下（文件名你定）。

  注意：不要编造未看到的内容；需要的话主动探索相关文件。

  覆盖能力：

  - 仓库探索（主动查找入口）
  - 文件阅读与归纳
  - 路径引用准确性
  - 文档产出能力

  ———

  ## Prompt 2（中阶能力：跨文件一致性+设计协同+可选 Skill/MCP）

  提示词：

  基于上一步的理解，输出一个“面对新人开发者的项目介绍网页”，并确保与文档内容一
  致：
  - 内容需包含：项目概览、核心能力、架构/流程、注意事项
  - 网页风格你自行决定（科技感/极简等均可）
  - 若你有 UI/UX 相关 Skill 可用，建议调用；若无也可自行完成
  - 可以使用mcp工具来进行搜索同类产品的功能点，在网页上进行对比

  注意：网页内容必须与上一步的文档一致，不得出现相互矛盾。

  覆盖能力：

  - 跨产物一致性维护（文档 ↔ 网页）
  - 设计/前端输出能力
  - Skill 触发与使用（可选）
  - MCP 扩展能力（可选）
  - 自主规划与执行

  ———

  ## Prompt 3（高阶能力：长上下文 + 轨迹自评 + 反思改进）

  提示词：

  请基于你刚才完成的两步任务，做一份“执行复盘与自评报告”：
  - 总结你完成任务的关键步骤与决策；
  - 分析有哪些地方做得好、有哪些不足（至少各 3 条）；
  - 如果系统有 trace / 日志 / 轨迹记录，请引用其中的证据来支持你的判断；
  - 给出 3 条可执行的改进建议（优先考虑稳定性与一致性）。

  注意：复盘要具体，不能只有泛泛的优缺点。

  覆盖能力：

  - 长上下文保持（引用前两步产物）
  - 自我评估与反思能力
  - trace/日志系统使用与分析
  - 可执行改进建议产出
