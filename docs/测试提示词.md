## Prompt 1（基础能力：探索+理解+小规模产出+mcp）

  提示词：

  请快速理解这个仓库是做什么的，并输出一份“项目概览”文档（可以使用subagent，如果有必要的话）：
  - 简要说明目标、核心模块、主要功能；
  - 引用真实文件路径作为依据；
  - 输出一个结构清晰的 Markdown 文件放在 demo/ 项下（文件名你定）。
  - 可以使用mcp工具来进行搜索同类产品如Claude Code、Codex Cli等的功能点，进行对比总结

  注意：不要编造未看到的内容；需要的话主动探索相关文件。

  覆盖能力：

  - 仓库探索（主动查找入口）
  - 文件阅读与归纳
  - 路径引用准确性
  - 文档产出能力

  ## Prompt 2（中阶能力：跨文件一致性+设计协同+ Skill）

  提示词：

  你需要分析自己的源代码，基于代码剖析的结果，创建一个面向用户的Agent自我介绍网页（以第一人称视角介绍自己）。
  - 请合理使用完成任务所需的所有工具，按照最优步骤执行
  - 内容与要求：
    - 可以使用mcp联网搜索获取同类竞品，分析优劣进行对比
    - 若你具备UI/UX相关技能（Skill），请调用并应用
    - 网页风格可自行选择（如玻璃拟态、拟物化、新拟态等均可）
    - 最终输出一个HTML文件，保存至demo/目录下（文件名可自定义）


  覆盖能力：

  - 跨产物一致性维护（文档 ↔ 网页）
  - 设计/前端输出能力
  - Skill 触发与使用
  - MCP 扩展能力
  - 自主规划与执行

  ———

  ## Prompt 3（高阶能力：长上下文 + 轨迹自评 + 反思改进）

  提示词：

  请基于你刚才完成的两步任务，做一份“执行复盘与自评报告”：
  - 总结你完成任务的关键步骤与决策；
  - 分析有哪些地方做得好、有哪些不足（至少各 3 条）；
  - 如果系统有 trace / 日志 / 轨迹记录，请引用其中的证据来支持你的判断；
  - 给出 3 条可执行的改进建议（优先考虑稳定性与一致性）。

  注意：复盘要具体，不能只有泛泛的优缺点。

  覆盖能力：

  - 长上下文保持（引用前两步产物）
  - 自我评估与反思能力
  - trace/日志系统使用与分析
  - 可执行改进建议产出
